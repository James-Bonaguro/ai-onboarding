# 04 — Voice Mode

## Objective

Voice Mode is a high-speed interface for thinking, not a replacement for written rigor.
Use it to accelerate exploration, then convert conclusions into structured text.

## Best use cases

Voice Mode performs well when you need to:

- interrogate an idea in real time,
- pressure-test assumptions quickly,
- map decision branches before drafting,
- rehearse explanations for stakeholders.

It is less suitable for final deliverables that require precise wording and citation control.

## Operating protocol

Use a two-stage workflow:

1. **Live reasoning (voice):** generate options, challenge assumptions, expose blind spots.
2. **Written consolidation (text):** lock structure, tighten claims, document uncertainty.

Do not skip stage two for consequential outputs.

## Voice interaction standards

During voice sessions, instruct the model to:

- interrupt weak logic politely,
- ask clarifying questions before concluding,
- separate hypothesis from evidence,
- summarize key takeaways every 5–10 minutes.

This prevents drift into conversational noise.

## Fast command patterns

Useful voice directives:

- “Challenge this thesis like a skeptical credit committee member.”
- “Give me three downside scenarios and their trigger indicators.”
- “What assumptions am I making without evidence?”
- “Summarize decisions, open questions, and next diligence steps.”

Short commands are effective when standards are already set.

## Risk controls

Voice can create false closure due to conversational fluency.
Mitigate this by requiring a written post-session summary with:

- provisional conclusions,
- confidence levels,
- unresolved data gaps,
- explicit next actions.

If it is not written and structured, it is not decision-ready.

## Bottom line

Voice Mode is excellent for cognitive velocity.
Written mode is essential for analytical accountability.
Use both deliberately.
