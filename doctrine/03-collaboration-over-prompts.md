# 03 — Collaboration Over Prompts

## Core claim

The highest-value use of ChatGPT is iterative collaboration, not prompt collection.
A static “best prompt” cannot replace an adaptive working conversation.

## The prompt-library trap

Prompt libraries feel productive because they create speed and repeatability.
But in complex knowledge work they break down fast:

- every deal context is different,
- assumptions shift as new data appears,
- useful outputs require follow-up scrutiny.

A one-shot prompt rarely captures dynamic uncertainty.

## Better model: managed dialogue

Run each task as a structured collaboration with checkpoints:

1. **Frame** — define objective, scope, and constraints.
2. **Draft** — generate a first pass with explicit assumptions.
3. **Interrogate** — challenge logic, missing data, and alternative interpretations.
4. **Refine** — tighten for audience, format, and decision use.
5. **Finalize** — produce output and unresolved questions.

The model’s value compounds across this cycle.

## Behavioral rules

Treat the model like a junior analyst with massive breadth and uneven judgment:

- ask for rationale, not just conclusions,
- require confidence tagging,
- challenge leaps in logic,
- rerun sections that fail your standard.

Do not reward speed over integrity.

## Collaboration patterns that work

- “Give me two competing theses and the evidence threshold that would falsify each.”
- “Where am I likely anchoring too early?”
- “Rewrite this for IC memo standards: tighter claims, explicit caveats, no rhetorical inflation.”

These patterns produce higher signal than “improve this prompt.”

## Bottom line

Your edge is not having the best prompt.
Your edge is running a rigorous analytic dialogue that converges on defensible output.
