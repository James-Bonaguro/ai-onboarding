# 04 — Iteration Is the Skill

## Core claim

Quality in AI-assisted work is achieved through controlled iteration.
First outputs are drafts, not verdicts.

## Why first drafts fail

Even with good context, initial responses often contain:

- over-broad framing,
- unsupported assertions,
- missing edge cases,
- weak prioritization.

This is normal.
The mistake is treating the first version as final because it sounds complete.

## Iteration loop (professional grade)

Use a repeatable loop:

1. **Diagnose** — identify where output fails your standard (logic, evidence, structure, tone).
2. **Direct** — issue precise revision instructions tied to failure points.
3. **Differentiate** — request alternatives and trade-offs, not single rewrites.
4. **Decide** — select best elements and force synthesis into final form.

Iteration is not “ask again.”
It is targeted quality control.

## Revision prompts that increase signal

- “Tighten claims to only what is source-supported; move speculation to a separate section.”
- “Rank key risks by downside severity and probability, with rationale in one sentence each.”
- “Cut 30% of words while preserving analytical content.”
- “Identify where this argument would be attacked by a skeptical credit committee.”

Each instruction narrows ambiguity and improves output reliability.

## Stop criteria

End iteration when all are true:

- logic is coherent and testable,
- assumptions are explicit,
- uncertainties are visible,
- structure matches end use,
- language is concise and non-theatrical.

If one is missing, continue iterating.

## Bottom line

Prompting is table stakes.
Iteration discipline is the actual skill.
